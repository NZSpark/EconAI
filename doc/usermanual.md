# EconAI 用户使用手册

> 版本：v1.3 | 适用于 EconAI v1.3

---

## 1. 系统简介

EconAI 是一个 AI 驱动的经济政策分析工具包。您可以上传政策文献、研究报告等文档，系统自动完成文献综述、政策草案、政策比较、技术解读等分析任务，输出带**逐句来源追溯**的专业报告。

### 核心功能

| 功能 | 说明 |
|------|------|
| 文献综述 | 基于上传文献自动生成结构化的文献综述报告 |
| 政策草案 | 根据背景信息自动撰写政策文件草案 |
| 政策比较 | 多份政策文件横向对比分析 |
| 技术解读 | 对技术标准/法规进行条款解读和合规分析 |

### 引用追溯

所有 AI 生成的报告都带**逐句引用标注**，可追溯到原始文献的具体段落和页码。引用分为三个置信度等级：

| 颜色 | 置信度 | 含义 |
|------|--------|------|
| 绿色 | 直接引用 | 高置信度，内容与原文高度匹配 |
| 黄色 | 模糊引用 | 中等置信度，语义相关但非直接对应 |
| 红色 | 不确定 | 低置信度，推断性内容 |

## 2. 登录系统

### 2.1 打开系统

在浏览器中访问系统管理员提供的链接（例如 `https://econai.your-institution.cn`）。

### 2.2 登录

1. 输入您的**用户名**和**密码**
2. 点击「登录」按钮

首次登录请使用管理员分配的账号。默认管理员账号：
- 用户名：`admin`
- 密码：`Admin@123456`（首次登录后请修改）

登录成功后将自动跳转到项目列表页面。

### 2.3 登出

点击页面右上角的用户头像 → 选择「退出登录」。

---

## 3. 项目管理

### 3.1 查看项目列表

登录后进入**项目列表**页面，展示您有权限查看的所有项目。

- **搜索**：在搜索框输入项目名称后按回车或点击「搜索」
- **状态过滤**：切换「活跃」/「已归档」查看不同状态的项目
- **分页**：使用底部分页控件浏览

### 3.2 创建项目

1. 点击右上角「创建项目」按钮
2. 填写表单：
   - **项目名称**：例如"2024数字贸易政策研究"
   - **描述**：项目背景和目标的简要说明（可选）
   - **所属项目组**：输入项目组 ID
3. 点击「创建」

### 3.3 进入项目

点击项目名称或「查看」按钮进入项目详情页。项目详情页包含两个子页面：

- **知识库**：管理项目文档
- **任务**：管理分析任务

### 3.4 归档项目

对不再活跃的项目，点击「归档」按钮。归档后项目变为只读，不再出现在默认列表中。

## 4. 知识库管理

知识库是您项目的"资料库"，所有上传的文档都会在这里被解析、分块、索引，供 AI 分析时检索使用。

### 4.1 上传文档

在项目详情页的「知识库」标签中：

1. **拖拽上传**：将文档文件拖到上传区域
2. 或点击上传区域选择文件
3. 支持的文件格式：

| 格式 | 扩展名 |
|------|--------|
| PDF | `.pdf` |
| Word | `.docx`, `.doc` |
| Excel/CSV | `.xlsx`, `.xls`, `.csv` |
| PowerPoint | `.pptx`, `.ppt` |
| Markdown/文本 | `.md`, `.txt` |
| 邮件 | `.eml` |
| 网页 | `.html`, `.htm`, `.mhtml`, `.mht` |
| 图片（OCR） | `.png`, `.jpg`, `.jpeg`, `.tiff`, `.bmp`（文字内容自动识别） |

- 单个文件最大 100MB
- 上传后显示进度条，完成后自动开始解析

> **OCR 图片识别**：系统不仅支持单独上传的图片文件，还会自动提取 PDF、DOCX、PPTX 和 HTML 文档中嵌入的图片，并对图片中的文字进行 OCR 识别，识别结果会自动合并到文档的文本内容中。

### 4.2 查看文档列表

文档列表显示所有已上传的文档，包含：
- 文件名、格式、文件大小
- 解析状态：`pending`（等待中）、`parsing`（解析中）、`ready`（就绪）、`error`（解析失败）
- 操作按钮：**下载**（下载原始文件）、**详情**、**重索引**、**删除**

可按状态过滤文档列表。

### 4.3 下载文档

在文档列表的「操作」列点击**下载**按钮，浏览器将自动下载原始文档文件。

### 4.4 查看文档详情

点击文档行可查看详情面板：
- 元数据（标题、作者、日期、来源、页数等）
- 解析状态详情
- 如果解析失败，可点击「重新索引」按钮重试

### 4.5 删除文档

点击文档行末的「删除」按钮，确认后文档及其所有关联数据将被级联删除。

### 4.6 搜索知识库

在搜索框中输入关键词，系统进行混合检索（语义匹配 + 关键词匹配）：
- 结果列表显示匹配的文本片段
- 每个结果标注**来源文档完整文件名**（含扩展名，如"ENG60459511 - Approved Engineering Plan.pdf"）和相关性分数
- 高亮显示匹配的关键词

## 5. 分析任务

### 5.1 创建任务

在项目详情页的「任务」标签中：

1. 点击「创建任务」按钮
2. 选择任务类型：

| 类型 | 用途 | 适用场景 |
|------|------|----------|
| 文献综述 | 对已有文献进行系统梳理和总结 | 研究背景调研 |
| 政策草案 | 撰写政策文件草案 | 政策制定 |
| 政策比较 | 对比分析不同政策文件 | 政策评估 |
| 技术解读 | 解读技术标准/法规条款 | 合规分析 |

3. 填写任务表单：
   - **标题**：任务名称，例如"数字贸易壁垒文献综述"
   - **描述**：任务的详细说明和要求
   - **知识源**：选择使用哪些文档作为分析依据
   - **输出格式**：选择输出报告格式（docx/md/xlsx/pptx）
   - **分析参数**：根据任务类型配置具体参数
4. 点击「提交」启动分析

### 5.2 查看任务列表

任务列表显示所有分析任务，包含：

- 任务标题
- 任务类型（文献综述/政策草案/政策比较/技术解读）
- 状态：`pending`（等待）、`running`（执行中）、`completed`（已完成）、`failed`（失败）、`cancelled`（已取消）
- 进度信息
- 创建时间

可以按状态过滤任务，或使用分页浏览。

### 5.3 监控任务进度

创建任务后，系统自动开始分析。任务进度实时展示：

1. **步骤进度条**：显示当前执行步骤
   - 计划 → 检索 → 生成 → 校验 → 格式化 → 导出
2. **当前步骤描述**：显示正在执行的具体操作
3. **预估剩余时间**：根据步骤估算

进度每 3 秒自动刷新，任务完成后自动停止轮询。

### 5.4 任务失败处理

如果任务执行失败，页面会显示错误信息。您可以：
- 查看详细错误原因
- 点击「重试」重新执行任务

## 6. 查看分析结果

任务完成后自动进入**输出页面**。

### 6.1 预览报告

默认以 **Markdown 预览** 模式展示报告内容：
- 章节标题、正文、表格完整渲染
- 引用角标 `[1]`、`[2]` 等可点击查看详情

点击「原文」标签可切换到 Markdown 原文视图。

### 6.2 查看引用来源

点击报告中的引用角标，弹出**引用详情 Popover**，显示：

- 置信度标签（绿/黄/红）
- 来源文档名称
- 页码范围
- 原文摘录
- AI 生成的引用句子

### 6.3 引用列表面板

点击「引用列表」按钮打开侧栏面板：

- 按置信度过滤：全部 / 直接引用 / 模糊引用 / 不确定
- 每个引用卡片显示详细信息
- 顶部显示引用统计（总数、各置信度数量）

### 6.4 导出报告

在输出页面顶部选择导出格式，点击「导出」按钮：

| 格式 | 说明 |
|------|------|
| Markdown (.md) | 文本格式，适合进一步编辑 |
| Word (.docx) | GB/T 9704 公文标准格式，含完整排版 |
| Excel (.xlsx) | 对比分析表 + 引用清单 |
| PPT (.pptx) | 演示文稿格式 |

浏览器将自动下载导出的文件，文件名包含任务标题和中文支持。

## 7. 管理功能（管理员）

管理员可以访问后台管理功能。点击侧边导航栏的「管理」菜单进入。

### 7.1 用户管理

路径：`/admin/users`

- **查看用户列表**：显示所有注册用户
- **创建用户**：点击「创建用户」填写用户名、邮箱、密码和角色
- **编辑用户**：修改用户信息或角色
- **停用/启用用户**：切换用户账号状态

系统角色：

| 角色 | 权限 |
|------|------|
| 分析员 (analyst) | 查看项目、创建任务、查看输出 |
| 高级研究员 (senior_researcher) | 分析员权限 + 创建项目、管理知识库 |
| 项目管理员 (project_admin) | 高级研究员权限 + 管理项目组成员 |
| 系统管理员 (system_admin) | 全部权限 + 用户管理、审计日志 |

### 7.2 项目组管理

路径：`/admin/groups`

- **查看项目组列表**：显示所有项目组，含成员数
- **创建项目组**：填写名称和描述
- **管理成员**：点击「管理成员」按钮打开弹窗
  - **成员列表**：表格显示所有成员的**用户名**、**显示名称**、**角色**
  - **添加成员**：在搜索框中**输入用户名或显示名称**模糊搜索，从下拉列表中选择用户后点击「添加成员」
  - **移除成员**：点击表格右侧的「移除」按钮，确认后移除
- 搜索框只列出**尚未加入该组**的活跃用户，避免重复添加

项目是挂载在项目组下的，用户可以访问其所属项目组的全部项目。

### 7.3 审计日志

路径：`/admin/audit-logs`

- 查看系统中的所有操作记录
- 按**时间范围**过滤
- 按**操作用户**过滤
- 按**操作类型**（创建/修改/删除/查看）过滤
- 分页浏览历史记录

日志记录内容：操作人、操作时间、操作类型、目标资源、操作详情。

## 8. 典型工作流程

### 场景一：撰写文献综述

1. **创建项目**：新建项目"2024数字贸易政策研究"
2. **上传文献**：将相关 PDF/Word 文献拖入知识库上传区域
3. **等待解析**：等文档状态变为"就绪"（通常 1-2 分钟）
4. **创建任务**：选择「文献综述」类型，设置分析范围
5. **等待分析**：系统自动检索、生成、验证引用（通常 5-10 分钟）
6. **查看报告**：检查自动生成的结构化综述
7. **审查引用**：通过引用列表检查每个论点的来源可靠性
8. **导出**：导出为 Word 文档或 Markdown 文本

### 场景二：政策对比分析

1. **上传两份政策文件**到知识库
2. **创建任务**：选择「政策比较」类型
3. **选择对比维度**：在任务参数中指定关注的维度
4. **等待分析完成**
5. **查看对比矩阵**：系统生成多维度对比表
6. **导出 Excel**：导出为 .xlsx 获取对比分析报告

## 9. 常见问题

### 上传文档后长时间显示"解析中"

- PDF 文档页数较多时解析时间较长
- 包含大量嵌入图片的文档（如 PPTX、带图 PDF）需要额外的 OCR 识别，耗时可能更长
- 图片 PDF 需要整页 OCR 识别，可能耗时更长
- 如果超过 10 分钟仍未完成，可尝试删除后重新上传

### 任务执行失败

常见原因：
- Claude API 密钥未配置或额度不足（联系管理员）
- Claude API 自定义端点配置错误（Docker 内 `localhost` 需改为 `host.docker.internal`）
- 知识库中缺少相关文档（先上传文档再创建任务）
- 任务超时（大项目可能需要更长处理时间，重试通常可解决）

### 引用显示为"不确定"

- 表示 AI 生成的句子无法在原文中找到对应内容
- 建议检查该句子的合理性，必要时手动修改
- 可通过「引用列表」过滤查看所有低置信度引用

### 忘记密码

请联系系统管理员重置密码。系统暂不支持自助密码重置。
